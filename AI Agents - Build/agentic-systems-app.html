üìö Fundamentals of AI Agents

What is an AI Agent?

An intelligent system that can independently plan, make decisions, execute tasks, and adapt to changes‚Äîminimizing human involvement.

The 4 Pillars of Agent Architecture

üß† LLM (The Brain)

A powerful model capable of reasoning, planning, and generating step-by-step outputs.

Examples: GPT-4o, Claude Sonnet 4, Gemini 2.5 Pro, Llama 4

üëÅÔ∏è Perception (Senses)

Ability to understand inputs from multiple sources.

Types: Text, Images, Audio, Video, Chain of Thought

üõ†Ô∏è Tools (Hands)

External capabilities that allow agents to act, fetch data, or control systems.

Examples: APIs, Vector Stores, Calculators, Web Search, Code execution

üìì Memory (Notebook)

Storage for past interactions and learnings.

Types: Short-term (session context) and Long-term (vector DBs, SQL, file store)

Types of AI Agents

1. Self-Directed AI Agents

Fully self-directed systems capable of making decisions and executing actions without human input.

Workflow: Define goal ‚Üí Perceive environment ‚Üí Plan next actions ‚Üí Execute tasks via APIs/tools ‚Üí Observe and learn from results ‚Üí Self-correct and improve

2. Collaborative Multi-Agent Systems

Multiple agents working together to solve complex tasks through communication and coordination.

Workflow: Assign roles to agents ‚Üí Share context and data ‚Üí Divide and parallelize sub-tasks ‚Üí Exchange results and feedback ‚Üí Merge outcomes ‚Üí Produce optimized final output

3. Cognitive AI Agents

Agents that simulate human-like reasoning using memory, logic, and contextual understanding.

Workflow: Perceive input ‚Üí Retrieve relevant memory/context ‚Üí Reason and infer ‚Üí Generate responses ‚Üí Evaluate correctness ‚Üí Store learnings for future tasks

4. Tool-Augmented AI Agents

Agents enhanced by external tools, APIs, and databases for extended capabilities.

Workflow: Receive task ‚Üí Identify required tools ‚Üí Connect via API or plugin ‚Üí Fetch or process external data ‚Üí Validate results ‚Üí Return enriched response

5. Reflective (Self-Improving) Agents

Agents that continuously learn from feedback and refine their own performance.

Workflow: Execute task ‚Üí Analyze outcome ‚Üí Identify improvement opportunities ‚Üí Adjust reasoning or planning ‚Üí Update memory and models ‚Üí Improve accuracy over time

7 Pillars of Agentic AI

1. Autonomy

Agents operate independently & initiate actions without human input.

Tools: AutoGen, CrewAI, LangGraph, OpenAgents

2. Goal-Directed Planning

Agents decompose objectives into tasks & adapt plans to complete them.

Tools: AutoGPT, BabyAGI, ReAct, LangChain

3. Communication & Collaboration

Agents interact with other agents or humans to complete shared tasks.

Tools: AutoGen, CrewAI, LangGraph, ChatDev

4. Reasoning & Decision Making

Agents apply logic & contextual understanding to make smart decisions.

Tools: GPT-4o, Claude 3 Opus, Mistral, Chain-of-Thought

5. Tool Use & Environment Interaction

Agents use tools, APIs, browsers, and code to complete tasks.

Tools: LangChain Toolkits, Function Calling, BrowserPilot

6. Memory & Learning

Agents store and recall past events, context, and knowledge.

Tools: LangChain Memory, MemGPT, Pinecone, ChromaDB

7. Safety, Alignment & Evaluation

Agents behave ethically, reliably, and within defined boundaries.

Tools: Guardrails AI, Constitutional AI, TruLens

üé® Design Patterns for Agentic AI

ReAct Agent

Single-Agent

Used by: Most AI Agent Products

A reasoning and acting framework where an agent alternates between reasoning (using LLMs) and acting (using tools like Google or email).

When to use: General purpose tasks requiring flexible decision-making and tool use

from langchain.agents import create_react_agent
from langchain_openai import ChatOpenAI
from langchain.tools import WikipediaQueryRun

llm = ChatOpenAI(model="gpt-4o")
tools = [WikipediaQueryRun()]

agent = create_react_agent(
    llm=llm,
    tools=tools,
    prompt="You are a helpful research assistant"
)

result = agent.invoke({"input": "What is quantum computing?"})
import { ReActAgent } from "@langchain/agents";
import { ChatOpenAI } from "@langchain/openai";

const model = new ChatOpenAI({ model: "gpt-4o" });
const agent = new ReActAgent({ model, tools: [] });

const result = await agent.invoke({
  input: "What is quantum computing?"
});

Multi-Agent Workflow

Collaborative

Used by: Gemini Deep Research

A collaborative system where multiple specialized agents work together to build a comprehensive output with greater precision than single architectures.

When to use: Complex tasks requiring specialized expertise, parallel processing

from crewai import Agent, Task, Crew

researcher = Agent(
    role='Researcher',
    goal='Gather comprehensive information',
    tools=[search_tool]
)

analyst = Agent(
    role='Data Analyst',
    goal='Analyze and synthesize findings',
    tools=[analysis_tool]
)

crew = Crew(
    agents=[researcher, analyst],
    tasks=[research_task, analysis_task]
)

result = crew.kickoff()

Agentic RAG

Advanced RAG

Used by: Perplexity

Agentic RAG involves AI agents retrieving and evaluating relevant data to generate context-aware and well reasoned output using memory and tools.

When to use: Question answering with citations, research tasks, fact-checking

from langchain.agents import create_agent
from langchain.tools.retriever import create_retriever_tool

retriever_tool = create_retriever_tool(
    retriever=vector_store.as_retriever(),
    name="knowledge_base",
    description="Search the knowledge base"
)

planner = create_agent(llm, [retriever_tool])

result = planner.invoke({
    "input": "Compare quantum computing approaches"
})

‚úÖ Best Practices & Common Mistakes

10 Mistakes to Avoid

  1. Skipping clear goal definition ‚Üí Agent handles random tasks
  2. Poor prompt experience ‚Üí Clunky, confusing agent
  3. No integration with external tools ‚Üí Limited usefulness
  4. Not using a vector database ‚Üí Agent doesn't recall
  5. Not using a database ‚Üí Agent starts fresh every time
  6. Overloading with too many tools ‚Üí Too much autonomy
  7. Foregoing modular design ‚Üí Difficult to scale
  8. Ignoring evaluation metrics ‚Üí Can't measure performance
  9. No feedback loop ‚Üí Agent never improves
  10. Overlooking guardrails ‚Üí Harmful outputs

Model Selection Guide

Use non-reasoning LLMs for planning by default. Provide clear tools. Test prompts with different models. Balance cost, speed, and task accuracy.
